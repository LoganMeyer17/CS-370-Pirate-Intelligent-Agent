# CS-370-Pirate-Intelligent-Agent
CS 370 Project 2 

* What code were you given? What code did you create yourself?

For this project, I worked on creating a pirate intelligent agent that used reinforcement learning to find the treasure in a maze. The starter code already included the maze environment, the game experience class, and much of the project structure.
My job was to complete the deep Q-learning portion of the project. I created the code that trained the agent by selecting actions, updating the neural network, managing the training process, and improving the pirate's decisions over time.
Seeing the agent gradually learn the correct path made the project much more rewarding because it showed that the training was actually working.

* What do computer scientists do and why does it matter?

Computer scientists solve problems by designing software and systems that make tasks easier, faster, or more efficient. They analyze problems, develop solutions, and continue improving those solutions over time.
This work matters because software is used in almost every part of daily life, from healthcare and education to businesses and entertainment. The technology people rely on every day exists because computer scientists continue developing and improving it.

* How do I approach a problem as a computer scientist?

When I approach a problem, I try to understand the goal before worrying about the solution. After that, I break the problem into smaller pieces that are easier to solve. During this project, I spent a lot of time testing, making changes, and figuring out why the agent was or wasn't learning correctly. That process reminded me that solving problems usually takes patience and several rounds of testing before everything works the way you expect.

* What are my ethical responsibilities to the end user and the organization?

I believe computer scientists have a responsibility to create software that is reliable, secure, and fair for the people who use it. Organizations trust developers to build systems that protect data and work as intended, while users expect software to perform correctly and safely. As AI continues to become more common, developers also need to think about how their systems make decisions and whether those decisions could negatively affect users.
This course reinforced the importance of building software responsibly and thinking beyond just getting the code to work.
